# Team SOPs — the short version

> Rules we actually follow, explained with slightly too much personality.

---

**1. Daily updates in the tracker**

Log progress while it’s still in your head. Tomorrow-you is not a reliable witness.

**2. Notes in meetings**

Write things down. If it isn’t captured, it didn’t happen — and “I thought someone else wrote it” is not a strategy.

**3. Attention to detail**

The gap between “done” and “done?” is often one typo, one edge case, or one unchecked box. Be the person who catches it.

**4. Before you log off**

Check in with **Shaff** or **Hisan** so handoffs don’t turn into midnight surprises.

**5. AI for code review**

Run the diff past an AI first. It’s extra eyes before human pride enters the chat.

**6. Internal QA + AI testing**

Exercise the test set; use AI for checklists and automation where it helps. “Works on my machine” is not a release channel.

---

_Take notes in meetings. Yes, still._
